五、单行处理函数（一个输入对应一个输出）

1、常见单行处理函数：

lower 转换小写
  如：mysql> select lower(ename) as ename from emp;


upper 转换大写
  如：mysql> select upper(ename) as ename from emp;

substr 取子串，格式：substrate（被截取的字符串，起始下标，截取的长度）
下标都是从1开始的
  如：mysql> select substr(ename, 1, 1) as ename from emp;


length 取长度，格式：length(被取的字符串)
  如：mysql>  select length(ename) enamelength from emp; （空格表省略的 as ，为起别名）


concat 字符串拼接，格式：concat（拼接字符串1，拼接字符串2）
拼接后为12，中间无空格
如mysql>  select concat(empno,ename) from emp;

trim 去空格，格式：trim（）
其()内的所有空格都将去除


round 四舍五入，格式：round（数字1，数字2）
数字1为要四舍五入的数，数字2为小数点后保留几位数
如：round（1234.56，1）结果为1234.6
       round（1234.56，0）结果为1235
       round（1234.56，-1）结果为1240

rand（） 生成随机数，格式：rand（）
随机生成0~1的数字
如：随机生成0~100的整数


ifnull 转化null，格式：ifnull（A，B）
其中A为参与的那一列数据，将null转化为B（有null参与的任何计算结果都为null，如1+null=null，与1+0=1不同）


case 如果，格式：case ... when ... then ... else end
其中when ... then ... 的个数按需求给，不变的部分用else，类似于 C 的 if 语句

找出员工名字首字母为 A 的员工信息：
①：模糊查询
mysql> select ename from emp where ename like ‘A%’;
②：substr函数
mysql> select ename from emp where substr(ename,1,1) = ‘A’;

将名字首字母改为大写（原本全为小写）
运用了函数的嵌套
①select upper(substr(name,1,1)) from t_student; 取学生名字首字母并改为大写
②select substr（name,2,length(name) - 1） from t_student; 取学生学生去掉首字母的名字保持不变
③select concat（upper(substr(name,1,1))，substr（name,2,length(name) - 1)）as result from t_student;拼接两部分并起名为result，的学生名字首字母大写后的结果

六、分组函数（多个输入对应一个输出）

1、共五个（分组函数使用时需进行分组，否则默认将整张表视为一组）
count 计数
sum 求和
avg 平均值
max 最大值
min 最小值


2、小注意：
  ①分组函数自动忽略NULL，碰到null就自动跳过，不需要提前对null进行处理
  ②count（具体字段）：该字段下所有不为null的内容个数
      count（*）：统计表中的行数，一行只要有数据则count++
  ③分组函数不得直接用在where子句中
     错误：select ename,sal from emp where sal > min(sal);
  ④分组函数可以连着一起用
     select sum(sal),min(sal),max(sal),avg(sal),count(*) from emp;

七、分组查询
先进行分组，再对分组各自分别操作，格式：select … from … group by …；

多个语句结合，固定的输入顺序为：select … from … where … group by … order by …;该顺序不可调换
计算机执行顺序为：from–>where–>group by–>select–>order by

select ename,sal from emp where sal > min(sal);报错原因：分组函数需分组后才能使用，where 执行时还没分组，所以 where 后面不能使用分组函数
select sum(sal) from emp;看似没分组，实际上是将整张表默认分为一组，且 select 在 group by 后面执行，因此不会报错


下面的语句①从 emp 表中查询数据；②根据 job 字段进行分组；③对每一组进行 sum（sal）

！！！在一条 select 语句中，如果有 group by 语句，select后只能跟：参加分组的字段，以及分组函数，其他一律不加，否则报错，在其他数据库中也是如此

having 语句，对分组结果进一步过滤，其不能代替 where，也不能离开 group by 单独使用
